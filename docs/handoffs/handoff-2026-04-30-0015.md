## 8. Gotchas & Landmines

These ate hours each. Don't fall into them again:

1. **CSS scope bug — the worst one** — `HOMEPAGE_OVERRIDES` only injects on the homepage. Universal rules placed there don't apply to about-me or any project page. Symptom: gray IM Creator footer visible on about-me, oversized logo, blue underlined HOME/ABOUT links. Fix: move universal rules to `width_fix` in `fix_pages.py`. **Albert's "bro, what the fuck" moment was triggered by this.** Verify CSS rules are in the deployed page's portfolio-fix `<style>` block.
2. **`xprs.imcreator.com` is DEAD** — older Wayback HTML references it for CSS/JS. Without rewriting to `www.imcreator.com`, older pages render blank. `fix_pages.py` does this rewrite via simple string replace.
3. **Captured pages had `data-app-version=...-no-viewer`** — that disables IM Creator's layout engine bundle. Stripping `-no-viewer` matters for any page that DOES need the engine.
4. **Triple-logo bug** — re-running `fix_pages.py` with regex stripping left some prior injections behind. Now using bs4 `find_all(class_=).decompose()`.
5. **about-me text flicker** — IM Creator's runtime engine kept resetting margin between 0 and 20px. Fixed by stripping `spimeengine.js` + `all_js.js` only on about-me. Don't apply this fix universally — other pages may need the engine for layouts that aren't fully baked into Wayback HTML.
6. **Profile photo lost border-radius** — when we removed spimeengine on about-me, the engine no longer applied border-radius:50% to `#vbid-686774fa-nvkm78sz`. Now hardcoded in CSS.
7. **Inner-pic divs were empty placeholders** — IM Creator pages have `<div class="inner-pic">` placeholders that were SUPPOSED to be filled by JS at runtime. They render as gray boxes. Safe to remove. But the **profile photo on about-me is in a different element** (an `<img>` with class `magic-circle-holder`), not an `inner-pic` — make sure not to confuse them.
8. **SingleFile / browser save approaches don't work** — Albert tried earlier; the IM Creator editor doesn't fully render the content into the saved DOM. The JS-console-snippet approach he eventually used works but requires manual run per page.
9. **GitHub Pages CDN cache lasts ~10 minutes** — after pushing, the live URL may serve stale content. Use `?cb=<timestamp>` query string when verifying changes, or hard-refresh.
10. **Browser HSTS cache for github.io** — when the GitHub Pages custom-domain field was briefly set to `changhsiju.xyz` and removed, browsers cached a permanent redirect. Symptom: `yabroexperiments.github.io/ceci-portfolio/` redirects to `changhsiju.xyz` even after the setting is gone. Fix: clear `chrome://net-internals/#hsts` for both domains.
11. **Tab disappearing in Chrome MCP** — Chrome MCP tabs get killed unpredictably. Always check `tabs_context_mcp` and re-create if needed. Don't get attached to a tab ID.
12. **Captured-page nested page-box parsing** — each `gallery-box` contains multiple `page-box` children, each with its own image+caption. The first version of `parse_section` collapsed them into one big section, losing per-image captions. Don't simplify back to that — Ceci specifically wanted per-image captions.
13. **`@import` inside an inline `<style>` block** — must be at the very top of the block. Currently it's after a few rules; browsers tolerate it but it's spec-violating. Hasn't caused issues yet but could.
